# 账单查询系统

一站式管理信用卡账单、借记卡流水、证券交易记录、电商流水。

## 数据源

| 类型 | 来源 | 数量 |
|:----|:----|:----:|
| 信用卡 | 14家银行（IMAP邮箱+坚果云PDF/XLS） | 8,701条 |
| 借记卡 | 多银行流水 | 1,953条 |
| 证券/基金 | 银河证券拖拉机交割单（Excel+TXT） | 10,000+条 |
| 电商 | 京东交易CSV（9个文件） | 2,558条 |

## 证券交易 - LOF基金套利知识

系统支持银河证券"拖拉机账户"的交割单管理。通过学习实践中发现了以下关键点：

### 拖拉机账户
- 同一个证券账户可开多个**股东账户**（深圳6个+上海1~4个）
- 各账户并行操作，数据几乎镜像一致
- **前导0问题**：深圳股东账户是10位含前导0（如`0105015768`）
  - Excel导出缺前导0（`105015768`）
  - TXT交割单正确（`0105015768`）
  - 必须统一为10位格式

### 深交所 vs 上交所 差异

| 项目 | 深交所(16xxxx) | 上交所(50xxxx) |
|:----|:-------------|:--------------|
| 申购记录 | `开放基金申购` 一笔记账（扣钱+到账合一） | `上证LOF申购`（扣钱）+ `股份转入`（到账）两笔 |
| 股东账户 | 0xxxxx（深圳股东账户） | Axxxxxxx（A股）+ Fxxxxxxx（基金） |
| 持仓计算 | 按stock_balance逐账户取最新>0汇总 | 同上 |

### 套利流程

**场内申购套利（T+1~T+2）：**
```
开放基金申购(NAV) → T+1/T+2份额到账 → 证券卖出(市价)
```

**场外申购转场内套利（T+3~T+4）：**
```
OTC资金划出(无标的) × 多日 → 转托管入(份额到账) → 证券卖出(市价)
```

### T+N交收规则
| 品种 | 类型 | 交收 |
|:----|:----|:----:|
| 白银基金(161226) | 商品基金 | T+1 |
| 标普科技(161128) | QDII LOF | T+2 |
| 华宝油气(162411) | QDII LOF | T+2 |
| 美元债(501300) | 上交所LOF | T+2 |
| 场外申购转场内 | - | T+3~T+4 |

### 持仓计算算法
1. 按 `(stock_code, shareholder_account)` 分组取最新 `stock_balance`
2. 筛选 `stock_balance > 0` 的活跃记录
3. 按 `stock_code` 汇总SUM
4. 已实现盈亏 = 卖出生额 - 申购支出（排除当前持仓部分）
